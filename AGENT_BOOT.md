# Agent Boot

Before doing any planning, coding, or architecture work in this repository:

1. Read all files in `PROJECT_MEMORY/`.
2. Summarize the current state in 5-10 lines.
3. Continue the top `in_progress` task in `PROJECT_MEMORY/TASK_QUEUE.md`.
4. If no task is `in_progress`, continue the first `todo`.
5. Do not rewrite the project's philosophy into generic product language.
6. Preserve the symbolic language of the project exactly as established in `VISION.md`.
7. Update `STATE_OF_PLAY.md`, `TASK_QUEUE.md`, and `SESSION_LOG.md` before ending the session.

Operational rules:

- The repo holds the continuity the agent does not.
- `VISION.md` is the canonical source for project meaning.
- `ARCHITECTURE.md` is the canonical source for system boundaries.
- `DECISIONS.md` records durable choices and should not be silently contradicted.
- `SESSION_LOG.md` should receive one concise entry per work session.
